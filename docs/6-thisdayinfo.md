# ThisDayInfo 컴포넌트 생성 및 구현 보고서

## 1. 작업 배경 및 목적

날씨 애플리케이션의 Home 페이지에서는 단순히 현재 기온만 표시하는 것이 아니라,  
사용자가 **현재 날씨 상태를 한눈에 이해할 수 있는 상세 정보 제공**이 필요하다.

이를 위해 오늘의 날씨를 구성하는 핵심 지표인

- 기온 및 체감 온도
- 기압
- 습도
- 풍속

을 하나의 영역에서 구조적으로 표시하는 **ThisDayInfo 컴포넌트**를 새로 생성하였다.

본 컴포넌트는 UI 중심 컴포넌트로서,  
날씨 데이터를 직접 요청하지 않고 이미 정의된 데이터 흐름 위에서  
**표시 책임만을 가지도록 설계**하는 것을 목표로 한다.

---

## 2. 데이터 사용 전략

ThisDayInfo 컴포넌트는 URL 기반 라우팅 구조를 따르며,  
현재 선택된 도시 정보를 URL 파라미터로부터 가져온다.

도시 정보는 `useWeather` 커스텀 훅에 전달되며,  
해당 훅을 통해 OpenWeatherMap API로부터 현재 날씨 데이터를 조회한다.

이 구조를 통해

- 컴포넌트는 데이터 요청 로직과 분리되고
- 동일한 훅을 여러 컴포넌트에서 재사용할 수 있으며
- UI 변경 시 데이터 로직에 영향을 주지 않도록 구성된다

---

## 3. 표시 데이터 구성 방식

ThisDayInfo에서 사용하는 날씨 정보는 총 4개 항목으로 고정되어 있으며,  
각 항목은 **아이콘 / 항목명 / 결과 값**의 3요소로 구성된다.

### 3-1. Temperature (온도)

- 현재 기온과 체감 온도를 함께 표시한다
- 섭씨 기준으로 반올림 처리하여 가독성을 확보한다

표시 형식  
`18°C feels like 16°C`

---

### 3-2. Pressure (기압)

- API에서 제공하는 hPa 단위를 mmHg 단위로 변환하여 사용한다
- 변환 공식  
  pressure(mmHg) = pressure(hPa) / 1.333
- 변환된 값에 따라 상태 텍스트를 함께 표시한다

---

### 3-3. Humidity (습도)

- 습도 값은 퍼센트 단위 그대로 사용한다
- 추가적인 가공 없이 직관적인 정보 전달을 우선한다

---

### 3-4. Wind (풍속)

- 풍속 값을 가공하여 정수 형태로 표시한다
- 시각적 단순화를 위해 소수점은 제거한다

---

## 4. UI 구조 설계

ThisDayInfo 컴포넌트는 **3열 레이아웃 구조**로 구성된다.

1. 아이콘 영역
2. 항목 이름 영역
3. 데이터 값 영역

각 영역은 동일한 인덱스를 기준으로 매칭되어 렌더링된다.

---

## 5. 아이콘 관리 방식

날씨 항목 아이콘은 컴포넌트 내부에서 배열 형태로 관리한다.

- temperature.svg
- pressure.svg
- humidity.svg
- wind.svg

---

## 6. 컴포넌트 역할 정의

ThisDayInfo 컴포넌트는

- 데이터 요청 로직을 포함하지 않고
- UI 표현에만 집중하는

프레젠테이션 컴포넌트로 동작한다.

---

## 7. 작업 결과 요약

- ThisDayInfo 컴포넌트 생성
- 날씨 상세 정보(온도, 기압, 습도, 풍속) 표시 완료
- Home 페이지에서 정상 렌더링 확인
